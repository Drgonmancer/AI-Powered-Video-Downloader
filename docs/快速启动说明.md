## 方式一：一键启动（Windows 推荐）

双击项目根目录 **`一键一键启动.bat`**（内部调用 `start.ps1`）。

或手动：

```powershell
cd <项目根目录>
powershell -ExecutionPolicy Bypass -File .\start.ps1
```

完成后在 **Edge / Chrome** 打开：`http://127.0.0.1:9000/`（勿用 Cursor 内置预览）。

---

## 方式二：开发模式（前后端分离调试）

### 步骤1：启动后端

```bash
cd backend
pip install -r requirements.txt
pip install yt-dlp httpx
uvicorn main:app --reload --port 9000
```

启动后访问 http://127.0.0.1:9000/docs 可看到Swagger API文档。

### 步骤2：启动前端（如果已有前端代码）

```bash
cd frontend
npm install
npm run dev
```

前端开发服务器默认在 http://localhost:3000，已配置代理转发到后端 `http://127.0.0.1:9000`。

## 验证后端是否正常工作

### 1. 健康检查

```bash
curl http://127.0.0.1:9000/api/health
# 返回: {"status":"ok"}
```

### 2. 测试解析接口 - YouTube示例

```bash
curl -X POST http://127.0.0.1:9000/api/parse \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ"}'
```

### 3. 测试解析接口 - 抖音视频 ⭐（v2.0新功能）

```bash
# 测试标准格式
curl -X POST http://127.0.0.1:9000/api/parse \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.douyin.com/video/7640844002783271851"}'

# 测试短链接（自动重定向解析）
curl -X POST http://127.0.0.1:9000/api/parse \
  -H "Content-Type: application/json" \
  -d '{"url": "https://v.douyin.com/i2KQBE8c/"}'
```

**预期返回**：
- `platform`: "douyin"
- `formats`: 包含3种格式（无水印、有水印、音频）
- `thumbnail`: 封面图URL
- `_direct_url`: 无水印视频地址

### 4. 测试下载接口 - 抖音无水印视频 ⭐

```bash
# 下载无水印版本（推荐）
curl -X POST http://127.0.0.1:9000/api/download \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://www.douyin.com/video/7640844002783271851",
    "format_id": "best",
    "output_format": "mp4"
  }'

# 下载仅音频
curl -X POST http://127.0.0.1:9000/api/download \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://www.douyin.com/video/7640844002783271851",
    "format_id": "audio",
    "output_format": "mp3"
  }'
```

返回 `{"task_id": "xxx", "status": "queued"}` 即成功创建任务。

### 5. 测试图片代理接口 ⭐

```bash
# 先获取一个抖音封面URL，然后测试代理
curl "http://127.0.0.1:9000/api/proxy-image?url=https%3A%2F%2Fp3-pc-sign.douyincdn.com%2Ftest.jpg" \
  --output test_image.jpg
```

### 6. 查看任务列表

```bash
curl http://127.0.0.1:9000/api/downloads
```

### 7. 查看设置

```bash
curl http://127.0.0.1:9000/api/settings
```

## 常见问题排查

### Q1: 抖音解析失败？

**可能原因及解决方案**：

1. **网络问题**
   ```bash
   # 检查是否能访问抖音域名
   ping www.iesdouyin.com
   ```

2. **视频已被删除或设为私密**
   - 错误信息：`所有解析方法均失败`
   - 解决方案：检查视频是否可以正常访问

3. **短链接失效**
   - 错误信息：`Short URL resolution failed`
   - 解决方案：尝试使用完整的douyin.com链接

### Q2: 如何确认抖音模块正在工作？

查看后端日志输出：
```
[Parser] Using specialized Douyin scraper (no Cookie required)
[DouyinScraper] Trying method 1b: Share Page...
[DouyinScraper] ✓ Success via Share Page! Title: xxx, Formats: 3
```

如果看到以上日志，说明抖音模块正常工作。

### Q3: 封面图无法显示？

确保使用图片代理接口：
```vue
<img :src="`/api/proxy-image?url=${encodeURIComponent(thumbnailUrl)}`" />
```

直接访问抖音CDN的图片会被浏览器跨域限制拦截。
