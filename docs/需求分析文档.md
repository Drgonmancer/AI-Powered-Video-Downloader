# AI万能视频下载器 - 项目技术文档

> **仓库归档说明**：开源仓库目录为 `backend/`、`frontend/`、`docs/`。技术文档统一放在 `docs/` 目录（与 GitHub 仓库 `liyupi/free-video-downloader` 结构一致）。

## 1. 项目概述

### 1.1 产品定位
一款本地运行的多平台视频下载工具，以Web应用形式交付，支持YouTube、Bilibili、抖音、快手、Twitter、Instagram等主流平台的一键下载。

### 1.2 产品形态
本地Web应用：用户启动后端服务，通过浏览器访问 `http://127.0.0.1:9000` 使用。

### 1.3 核心价值
- **一键解析**：粘贴链接即可自动识别平台、解析最佳质量
- **多平台覆盖**：基于yt-dlp引擎，支持1000+网站
- **批量高效**：播放列表/频道/收藏夹一键批量下载
- **专业管理**：队列控制、断点续传、速度限制、格式转换

│       │            │              │          │

**MVP原则**：
- 零外部依赖（除yt-dlp和ffmpeg外）
- 无数据库，任务状态全部在内存中维护
- 设置用内存字典存储（重启恢复默认值）
- 历史记录不持久化，仅当前会话可见

## 3. 功能模块详细设计（MVP）

### 3.1 视频解析模块

#### API设计
```
POST /api/parse
请求体: { "url": "https://www.youtube.com/watch?v=xxx" }
响应体: {
  "title": "视频标题",
  "description": "视频描述",
  "thumbnail": "封面图URL",
  "duration": 3600,
  "platform": "youtube",
  "uploader": "上传者",
  "formats": [
    { "format_id": "137", "ext": "mp4", "resolution": "1080p", "filesize": 1073741824 },
    { "format_id": "251", "ext": "webm", "resolution": "audio only", "filesize": 5242880 }
  ],
  "is_playlist": false,
  "playlist_count": null,
  "best_format": {                    // 智能推荐最佳格式
    "format_id": "137+251",
    "resolution": "1080p",
    "ext": "mp4",
    "note": "最佳画质合并流"
  }
}
```

#### 实现方式
```python
# 直接调用 yt-dlp --dump-json --no-download
import yt_dlp

def parse_url(url):
    ydl_opts = {'quiet': True, 'no_warnings': True}
    with yt_dlp.YoutubeDL(ydl_opts) as ydl:
        info = ydl.extract_info(url, download=False)
        return _normalize_info(info)
```

#### 智能推荐逻辑
- 优先选择MP4容器 + 最高分辨率
- 若无合并流，自动推荐 video_best + audio_best 组合
- 音频提取场景推荐最高音质音频流

### 3.2 下载模块

#### API设计
```
POST /api/download
请求体: {
  "url": "https://...",
  "format_id": "137+251",
  "output_format": "mp4",
  "quality": "best"
}
响应体: { "task_id": "uuid-xxx", "status": "queued" }

GET /api/downloads
响应体: [{ task_id, title, status, progress, speed, eta, ... }, ...]

POST /api/download/{task_id}/pause
POST /api/download/{task_id}/resume
DELETE /api/download/{task_id}

WebSocket /ws/downloads
实时推送进度更新
```

#### 内存任务管理
```python
# main.py 中直接使用 dict 存储
tasks: Dict[str, Task] = {}           # task_id -> Task对象

class Task:
    id: str
    url: str
    title: str
    status: str            # queued/downloading/paused/completed/failed
    progress: float = 0.0  # 0-100
    speed: str = ""
    eta: str = ""
    error: str = ""
    process: subprocess.Process | None = None
```

#### 进度回调
```python
def progress_hook(d):
    if d['status'] == 'downloading':
        tasks[current_task_id].progress = (
            d.get('downloaded_bytes', 0) / max(d.get('total_bytes', 1), 1) * 100
        )
        tasks[current_task_id].speed = d.get('_speed_str', '')
        tasks[current_task_id].eta = d.get('_eta_str', '')
        await ws_manager.broadcast(tasks[current_task_id].to_dict())
    elif d['status'] == 'finished':
        tasks[current_task_id].status = 'merging'
    elif d['status'] == 'completed':
        tasks[current_task_id].status = 'completed'
        tasks[current_task_id].progress = 100.0
```

### 3.3 批量下载模块

#### API设计
```
POST /api/batch
请求体: {
  "playlist_url": "https://...",      # 播放列表URL
  "quality": "best",
  "max_concurrent": 3
}
响应体: { "batch_id": "xxx", "total": 10 }
```

#### 实现
- yt-dlp原生支持播放列表下载，传入playlist URL即可
- 多个独立URL则循环创建Task加入队列
- 并发控制用 asyncio.Semaphore

### 3.4 格式转换模块

#### API设计
```
POST /api/convert
请求体: {
  "input_path": "/path/to/input.webm",
  "output_format": "mp4"
}
响应体: { "task_id": "uuid-xxx" }
```

#### 支持
- MP4/WebM/MKV 视频互转
- MP3/AAC/FLAC/WAV 音频提取

## 6. 启动流程

### 一键启动
```bash
一键启动.bat 或 start.ps1
```

自动完成：
1. 检查Python ≥ 3.11
2. pip install -r backend/requirements.txt
3. 检查yt-dlp，无则 `pip install yt-dlp`
4. 检查ffmpeg，无则提示安装
5. 创建 downloads/ 目录
6. 启动FastAPI（端口9000）
7. 自动打开浏览器

### 开发模式
```bash
# 后端
cd backend && uvicorn main:app --reload --port 9000

# 前端
cd frontend && npm run dev
```
