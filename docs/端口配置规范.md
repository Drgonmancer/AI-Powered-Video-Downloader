## 配置文件清单

### 1️⃣ 后端主程序 (main.py)
```python
port = int(os.environ.get("PORT", 9000))  # ✅ 后端端口
host = "0.0.0.0" if is_render else "127.0.0.1"
```

### 2️⃣ Vite 配置 (vite.config.js)
```javascript
server: {
  port: 3000,           // ✅ 前端开发服务器端口
  strictPort: false,    // ⚠️ 如果被占用会自动切换
  proxy: {
    '/api': {
      target: 'http://localhost:9000',  // ✅ 代理到后端
    },
    '/ws': {
      target: 'ws://localhost:9000',     // ✅ WebSocket代理
    }
  }
}
```

### 3️⃣ 环境变量 (.env.development)
```
VITE_API_URL=http://localhost:9000  # ✅ 后端地址
```

### 4️⃣ 启动脚本 (launcher.ps1)
```powershell
$BackendPort = 9000   # ✅ 后端端口
$FrontendPort = 3000  # ✅ 前端端口
```

### 5️⃣ Stripe 配置 (stripe_config.py)
```python
"success_url": "http://localhost:9000/payment/success",  # ✅
"cancel_url": "http://localhost:9000/pricing",            # ✅
```

## 常见问题

### Q1: 如何确认端口是否被占用？
**A:** 运行以下命令：
```powershell
netstat -ano | findstr ":9000 :3000"
```

### Q2: 如果新端口也被占用了怎么办？
**A:** 可以修改以下文件中的端口号：
- `launcher.ps1`: `$BackendPort` 和 `$FrontendPort`
- `vite.config.js`: `server.port`
- `main.py`: `os.environ.get("PORT", 9000)`

### Q3: 为什么 strictPort 设置为 false？
**A:** 为了提高容错性。如果指定端口被占用，系统仍能正常启动。
