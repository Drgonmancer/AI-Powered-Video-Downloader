## 一、背景与需求

### 1.1 业务需求

用户反馈：**"一定要支持抖音视频下载，而且不需要用户自己提供 Cookie"**

这是一个硬性需求，原因如下：

- **用户体验**：普通用户不知道如何获取Cookie
- **使用门槛**：提供Cookie对非技术人员来说太复杂
- **产品竞争力**：市面上大多数抖音下载工具都需要Cookie
- **合规性**：避免涉及用户隐私数据的收集

### 1.2 技术挑战

| 挑战 | 难度 | 说明 |
|------|------|------|
| ❌ 无Cookie认证 | ⭐⭐⭐⭐⭐ | 抖音API通常需要登录态 |
| ❌ 无水印提取 | ⭐⭐⭐⭐ | 需要找到无水印URL |
| ❌ 多URL格式支持 | ⭐⭐⭐ | 短链接、标准链接、精选页等 |
| ✅ 高清画质 | ⭐⭐ | 需要找到1080p源 |
| ✅ 视频封面 | ⭐⭐ | 需要从页面或API提取 |

### 1.3 最终实现的功能清单

✅ **核心功能**
- 无需任何用户配置（零门槛）
- 自动识别抖音链接（5种URL格式）
- 获取无水印高清视频
- 支持有水印版本（备用）
- 提取音频单独下载
- 自动提取视频封面图

✅ **增强功能**
- 多方法降级策略（提高成功率）
- 短链接自动解析
- 错误处理与日志记录
- 与现有系统无缝集成
- 前端组件适配（多格式选择器）

### 3.2 关键技术点

#### 3.2.1 公开API利用

抖音有一个公开的视频信息接口，不需要登录即可访问：

```
https://www.iesdouyin.com/share/video/{aweme_id}/
```

**特点**：
- 返回完整的视频元数据
- 包含播放地址列表
- 使用移动端UA可提高成功率
- HTML中嵌入了`window._ROUTER_DATA` JSON数据

#### 3.2.2 去水印原理 ⭐

抖音视频URL有两种形式：

```python

wm_url = "https://v26-web.douyinvod.com/playwm/xxx.mp4"
                                    ^^^^^^
                                  包含 playwm

# 无水印版本（替换后）
no_wm_url = wm_url.replace('playwm', 'play')
# → "https://v26-web.douyinvod.com/play/xxx.mp4"
                              ^^^^
                            替换为 play
```

**这就是整个方案的核心！一行代码搞定去水印！**

#### 问题1: iesdouxin API返回错误码11110 ⭐⭐⭐

**错误现象**：
```python
{
    "status_code": 11110,
    "status_msg": "参数错误"
}
```

**原因分析**：
- API需要加密签名参数（X-Bogus、_signature）
- 这些参数由JavaScript动态生成
- 直接调用会失败

**解决方案**：
❌ 尝试1：逆向JS算法生成签名
- 工作量巨大（2-3天）
- 抖音频繁更新算法
- 维护成本高

✅ 采用方案：切换到分享页面解析方法
- 不需要签名参数
- 直接从HTML中提取数据
- 成功率更高

**代码改动**：
```python
# 原方案（失败）
result = await parse_via_iesdouyin_api(aweme_id, normalized_url)

# 新方案（成功）⭐
result = await parse_via_iesdouyin_share_page(aweme_id)
```

**教训**：不要过度依赖需要签名的API，寻找更简单的替代方案。

#### 问题2: 短链接无法解析 ⭐⭐

**错误现象**：
```
Short URL resolution failed for https://v.douyin.com/i2KQBE8c/
```

**原因分析**：
- 短链接需要302重定向到完整URL
- httpx默认不跟随重定向
- 或者重定向次数限制

**解决方案**：

```python
async def resolve_short_url(url: str) -> str:
    """解析短链接"""
    headers = {
        'User-Agent': (
            'Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X) '
            'AppleWebKit/605.1.15 (KHTML, like Gecko) Version/16.0 '
            'Mobile/15E148 Safari/604.1'
        ),
    }

    async with httpx.AsyncClient(
        timeout=15,
        follow_redirects=True,  # 关键：跟随重定向
        headers=headers,
        verify=False
    ) as client:
        resp = await client.get(url)
        return str(resp.url)  # 返回最终的完整URL
```

**关键点**：
1. 使用移动端User-Agent
2. `follow_redirects=True` 跟随重定向
3. 从`resp.url`获取最终URL

#### 问题3: UnicodeEncodeError编码错误 ⭐

**错误现象**：
```
UnicodeEncodeError: 'gbk' codec can't encode character '\U0001f389' in position xx
```

**原因分析**：
- Windows系统默认GBK编码
- 视频标题包含emoji字符（🎉🔥💯等）
- GBK无法编码这些字符

**解决方案**：

```python
import re

def clean_text(text: str) -> str:
    """清理文本中的特殊字符"""
    # 替换常见emoji为空格或移除
    text = re.sub(r'[\U00010000-\U0010ffff]', '', text)
    # 移除其他可能导致问题的字符
    text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f-\x9f]', '', text)
    return text.strip()

# 使用示例
title = clean_text(item_info.get('desc', ''))
```

**扩展思考**：是否应该保留emoji？可以改为替换为文字描述。

#### 问题4: httpx参数错误 ⭐

**错误现象**：
```
TypeError: AsyncClient.get() got an unexpected keyword argument 'allow_redirects'
```

**原因分析**：
- httpx的`allow_redirects`是Client级别的参数
- 不能在`get()`方法中使用

**解决方案**：

```python
# 错误用法
client = httpx.AsyncClient(timeout=20)
resp = await client.get(url, allow_redirects=True)  # ❌

# 正确用法
client = httpx.AsyncClient(
    timeout=20,
    follow_redirects=True  # ✅ 在创建Client时设置
)
resp = await client.get(url)
```

**教训**：仔细阅读httpx文档，区分Client级别和方法级别的参数。

#### 问题5: JSON解析失败 ⭐⭐

**错误现象**：
```
json.decoder.JSONDecodeError: Expecting value: line 1 column 1
```

**原因分析**：
- HTML中的JSON被转义
- 包含大量`\uXXXX`转义序列
- 可能有嵌套的引号问题

**解决方案**：

```python
import json

def extract_json_from_html(html: str, start_idx: int) -> str:
    """从HTML中提取并清洗JSON数据"""
    # 找到JSON开始位置
    json_start = html.find('{', start_idx)

    # 找到匹配的结束位置（考虑嵌套）
    brace_count = 0
    in_string = False
    escape_next = False
    end_idx = json_start

    for i in range(json_start, len(html)):
        char = html[i]

        if escape_next:
            escape_next = False
            continue

        if char == '\\':
            escape_next = True
            continue

        if char == '"' and not escape_next:
            in_string = not in_string
            continue

        if not in_string:
            if char == '{':
                brace_count += 1
            elif char == '}':
                brace_count -= 1
                if brace_count == 0:
                    end_idx = i + 1
                    break

    raw_json = html[json_start:end_idx]

    # 清洗转义字符
    raw_json = raw_json.replace('\\"', '"')
    raw_json = raw_json.replace('\\\\', '\\')

    return raw_json
```

**关键点**：
1. 正确匹配花括号（考虑嵌套）
2. 处理字符串内的特殊字符
3. 清洗转义序列

## 六、关键代码解析

### 6.1 URL检测与标准化

```python
from urllib.parse import urlparse

DOUYIN_DOMAINS = ['douyin.com', 'iesdouyin.com']

def is_douyin_url(url: str) -> bool:
    """检测是否为抖音链接"""
    try:
        domain = urlparse(url).netloc.lower()
        return any(d in domain for d in DOUYIN_DOMAINS)
    except:
        return False


def normalize_douyin_url(url: str) -> str:
    """
    标准化为 video/{id} 格式
    支持多种输入格式：
    - https://www.douyin.com/video/123456
    - https://www.douyin.com/jingxuan?modal_id=123456
    - https://v.douyin.com/abc/
    """
    url = url.strip()

    # 如果已经是标准格式，直接返回
    if '/video/' in url and re.search(r'/video/\d+', url):
        return url

    # 尝试提取ID并重构
    aweme_id = extract_aweme_id(url)
    if aweme_id:
        return f'https://www.douyin.com/video/{aweme_id}'

    return url
```

**设计亮点**：
- 容错性强（try-except包裹）
- 支持多种格式输入
- 正则表达式精确匹配

### 6.2 视频ID提取

```python
import re

def extract_aweme_id(url: str) -> str:
    """从URL中提取视频ID"""
    patterns = [
        r'(?:video|modal_id|aweme_id)[/_:]([\d]+)',  # 标准格式
        r'/v/([a-zA-Z0-9]+)',                          # /v/ 格式
        r'modal_id=(\d+)',                             # query参数
        r'share/video/(\d+)',                          # 分享页格式
    ]

    for pattern in patterns:
        match = re.search(pattern, url)
        if match:
            return match.group(1)

    return ''
```

**为什么用多个正则？**
- 抖音URL格式多样
- 不同场景有不同的URL模式
- 多正则提高兼容性

### 6.3 主力解析方法 ⭐⭐⭐

这是整个模块的核心，约150行代码：

```python
async def parse_via_iesdouyin_share_page(aweme_id: str) -> dict:
    """
    通过分享页面解析视频信息
    成功率最高，推荐作为主力方法
    """

    share_url = f'https://www.iesdouyin.com/share/video/{aweme_id}/'

    # 关键：使用移动端User-Agent
    mobile_headers = {
        'User-Agent': (
            'Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X) '
            'AppleWebKit/605.1.15 (KHTML, like Gecko) Version/16.0 '
            'Mobile/15E148 Safari/604.1'
        ),
        'Accept-Language': 'zh-CN,zh;q=0.9,en;q=0.8',
        'Referer': 'https://www.douyin.com/',
    }

    async with httpx.AsyncClient(
        timeout=20,
        follow_redirects=True,
        headers=mobile_headers,
        verify=False  # 生产环境应设为True
    ) as client:
        resp = await client.get(share_url)

        if resp.status_code != 200:
            raise Exception(f'Share page returned status {resp.status_code}')

        html = resp.text

        # 提取JSON数据
        marker = 'window._ROUTER_DATA = '
        start_idx = html.find(marker)

        if start_idx == -1:
            raise Exception('Cannot find _ROUTER_DATA in page')

        payload = extract_json_from_html(html, start_idx + len(marker))

        try:
            router_data = json.loads(payload)
        except json.JSONDecodeError as e:
            raise Exception(f'Failed to parse JSON: {e}')

        # 提取视频信息（遍历所有可能的节点）
        loader_data = router_data.get('loaderData', {})

        for node_key, node_value in loader_data.items():
            video_info_res = node_value.get('videoInfoRes', {})
            item_list = video_info_res.get('item_list', [])

            if not item_list:
                continue

            item_info = item_list[0]  # 取第一个视频

            # === 提取视频URL ===
            video_data = item_info.get('video', {})
            play_addr = video_data.get('play_addr', {})
            play_urls = play_addr.get('url_list', [])

            if not play_urls:
                continue

            wm_url = play_urls[0]
            no_wm_url = wm_url.replace('playwm', 'play')  # ⭐ 去水印！

            # === 提取封面 ===
            cover = ''
            for key in ('cover', 'origin_cover', 'dynamic_cover'):
                cover_data = video_data.get(key, {}).get('url_list', [])
                if cover_data:
                    cover = cover_data[0]
                    break

            # === 提取音频 ===
            music_data = item_info.get('music', {})
            audio_urls = music_data.get('play_url', {}).get('url_list', [])
            audio_url = audio_urls[0] if audio_urls else ''

            # === 构建格式列表 ===
            formats = build_format_list(no_wm_url, wm_url, audio_url)

            # === 返回标准化结果 ===
            return {
                'title': clean_text(item_info.get('desc', '')),
                'thumbnail': cover,
                'duration': int(item_info.get('duration', 0) / 1000),  # ms → s
                'platform': 'douyin',
                'uploader': item_info.get('author', {}).get('nickname', ''),
                'formats': formats,
                'best_format': formats[0] if formats else None,
                '_direct_url': no_wm_url or wm_url,
                '_audio_url': audio_url,
            }

        raise Exception('No video found in response')
```

**代码亮点**：
1. **防御性编程**：每一步都有错误检查
2. **清晰的注释**：每个步骤都有说明
3. **容错处理**：遍历多个节点查找数据
4. **标准化输出**：与其他平台格式统一

### 6.4 多格式构建

```python
def build_format_list(no_wm_url, wm_url, audio_url):
    """构建多种下载格式选项"""
    formats = []

    # 格式1: 无水印高清（优先级最高）
    if no_wm_url:
        formats.append({
            'format_id': 'best',
            'ext': 'mp4',
            'resolution': '1080p',
            'filesize': 0,
            'vcodec': 'h264',
            'acodec': 'aac',
            'tbr': 0,
            'url': no_wm_url,
            'watermarked': False,  # 标记为无水印
        })

    # 格式2: 有水印高清（备用）
    if wm_url:
        formats.append({
            'format_id': 'wm_1080p',
            'ext': 'mp4',
            'resolution': '1080p (Watermark)',
            'filesize': 0,
            'vcodec': 'h264',
            'acodec': 'aac',
            'tbr': 0,
            'url': wm_url,
            'watermarked': True,  # 标记为有水印
        })

    # 格式3: 仅音频
    if audio_url:
        formats.append({
            'format_id': 'audio',
            'ext': 'mp3',
            'resolution': 'Audio Only',
            'filesize': 0,
            'vcodec': 'none',
            'acodec': 'aac',
            'tbr': 128,
            'url': audio_url,
            'watermarked': False,
        })

    return formats
```

**设计考量**：
- `watermarked`字段帮助前端显示不同UI
- 无水印格式排在第一位（推荐）
- 每个格式都有独立的URL（可直接下载）
