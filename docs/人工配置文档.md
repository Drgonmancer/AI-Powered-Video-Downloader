# 人工配置文档

> 运行项目前需要确认的环境配置项

---

## 一、必须安装的软件

### 1. Python (3.10+)

**检查方式**:
```bash
python --version
# 或
python3 --version
```

**安装地址**: https://www.python.org/downloads/

**Windows注意**: 安装时务必勾选 **"Add Python to PATH"**

---

### 2. ffmpeg

**检查方式**:
```bash
ffmpeg -version
```

**安装方式（Windows）**:

1. 访问 https://www.gyan.dev/ffmpeg/builds/ （推荐）
2. 下载 `ffmpeg-release-essentials.zip`
3. 解压到任意目录，如 `C:\ffmpeg`
4. 将 `C:\ffmpeg\bin` 添加到系统 **PATH 环境变量**
5. **重启终端/IDE** 后验证: `ffmpeg -version`

**或使用 winget 安装**:
```bash
winget install ffmpeg
```

**影响范围**: ffmpeg未安装时，格式转换功能不可用。视频下载功能不受影响。

## 二、自动安装的依赖（无需手动操作）

以下依赖由 `一键启动.bat` / `start.ps1` 或手动执行 pip 自动安装：

| 包名 | 用途 | 版本要求 |
|------|------|----------|
| fastapi | Web框架 | >=0.110 |
| uvicorn | ASGI服务器 | >=0.27 |
| yt-dlp | 视频下载引擎 | latest |
| websockets | WebSocket支持 | >=12 |
| pydantic | 数据校验 | >=2.6 |

**手动安装命令**（如果不用一键启动）:
```bash
cd backend
pip install -r requirements.txt
pip install yt-dlp
```

## 三、可选配置

### 3.0 AI 视频摘要（DeepSeek，解析时自动生成）

解析链接后会调用 **DeepSeek** 生成摘要，帮助你在下载前确认视频内容。

**本地配置文件**（已加入 `.gitignore`，不会提交到 Git）：

```
backend/ai_config.py
```

**首次使用**（任选其一）：

1. 复制 `backend/ai_config.example.py` 为 `backend/ai_config.py`（`ai_config.py` 已加入 `.gitignore`）
2. 手动复制：
   ```bash
   copy backend\ai_config.example.py backend\ai_config.py
   ```

**Stripe（会员支付）**：复制 `backend/config/stripe_config.example.py` → `backend/config/stripe_config.py`，或配置环境变量 `STRIPE_*`（见 [MEMBERSHIP_DEPLOYMENT_GUIDE.md](./MEMBERSHIP_DEPLOYMENT_GUIDE.md)）。

然后在 `ai_config.py` 中填写：

```python
DEEPSEEK_API_KEY = "sk-你的密钥"
```

`DEEPSEEK_BASE_URL` 与 `DEEPSEEK_MODEL` 已在项目中固定，**无需修改**：

| 配置项 | 固定值 |
|--------|--------|
| Base URL | `https://api.deepseek.com/v1` |
| 模型 | `deepseek-v4-flash` |

**获取 Key**：https://platform.deepseek.com/api_keys

**关闭摘要**：在 `ai_config.py` 中将 `AI_SUMMARY_ENABLED` 设为 `False`。

**安全提示**（上线 GitHub 前请确认）：

- `backend/ai_config.py`、`backend/config/stripe_config.py` 已加入 `.gitignore`，勿提交真实密钥
- 仓库内仅保留 `*.example.py` 模板；本地从模板复制后填写
- Stripe / JWT / DeepSeek 均使用环境变量或本地未跟踪配置文件

### 3.1 网络代理（如需访问YouTube等）

**方法A：环境变量**（推荐）
```bash
# Windows CMD
set HTTP_PROXY=http://127.0.0.1:7890
set HTTPS_PROXY=http://127.0.0.1:7890
powershell -File start.ps1

# Windows PowerShell
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
powershell -File start.ps1
```

**方法B：yt-dlp配置文件**

在用户主目录创建配置文件：

- **Windows**: `%APPDATA%\yt-dlp\config.txt`
- **Linux/Mac**: `~/.config/yt-dlp/config.txt`

内容示例：
```
--proxy http://127.0.0.1:7890
```

### 3.2 下载目录

默认下载到项目根目录下的 `downloads/` 文件夹。

可通过设置API修改：
```bash
PUT /api/settings
{"download_path": "D:/MyVideos"}
```

或在启动前修改 `backend/main.py` 中 config 的 `download_path` 值。

### 3.3 并发数

默认同时下载3个文件。可在运行时通过设置API调整（1-5）。
并发过高可能导致被目标网站限速/封IP。

### 3.4 速度限制

默认不限速。可通过设置API限制每秒下载字节数：
- 0 = 不限速
- 1048576 = 1MB/s
- 5242880 = 5MB/s
- 10485760 = 10MB/s

## 四、常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `ModuleNotFoundError: No module named 'yt_dlp'` | 未安装yt-dlp | `pip install yt-dlp` |
| `ffprobe/ffmpeg not found` | 未安装ffmpeg | 参考上方ffmpeg安装步骤 |
| 解析超时/失败 | 网络不通或被封 | 配置代理；更新yt-dlp (`pip install -U yt-dlp`) |
| 下载速度为0 | 目标网站限流 | 降低并发数或添加代理 |
| 中文文件名乱码 | 终端编码问题 | yt-dlp已内置处理，确保Python为UTF-8模式 |
| 端口9000被占用 | 其他程序占用该端口 | 修改main.py中的端口号 |

---

## 五、yt-dlp 更新维护

yt-dlp 需要定期更新以保持对各平台的支持：

```bash
pip install -U yt-dlp
```

建议每周更新一次。遇到解析失败时首先尝试此命令。
